关于Git常用的指令

Git版本库的创建

$git init;

通过这个命令把这个目录变为git可管理的仓库，执行之后，可以发现多一个.git的文件夹

$git add .;

将所有文件这里只是添加到暂存区

$git commit -m "commit";

这里是将暂存区的所有内容提交到master所在的分支，并且有附带介绍的信息

$git status;

掌握仓库当前的状态

$git diff;

展示修改的具体内容

$git log ;

记录文件的每一次改动的详细信息

$git log --oneline;

将每一次改动的详细信息以行的形式展示

$git  reset --hard HEAD^;

查看上一次的修改

$git  reset --hard HEAD~2;

查看上上一次的修改

撤销版本修改

1.在工作区进行修改，没有添加到暂存区

$ git checkout -- readme.txt

2.在工作区进行修改，并且执行了添加到暂存区的操作，但是没有提交到版本库

$ git reset HEAD readme.txt ；//在此处回到情况1
$ git checkout -- readme.txt

3.修改后提交到了版本库

$ git reset --hard HEAD^

分支管理

首先，我们创建dev分支，然后切换到dev分支：

$ git checkout -b dev
Switched to a new branch 'dev'

然后，用git branch命令查看当前分支（所有分支）：

$ git branch
* dev
  master

之后可以在这个分支上面修改文件，然后添加到版本库：

$ git add readme.txt
$ git commit -m 'add text about....'

dev分支的工作完成，我们就可以切换回master分支：

$ git checkout master
Switched to branch 'master'

现在，我们把dev分支的工作成果合并到master分支上：

$ git merge dev

合并完成后，就可以放心地删除dev分支了：

$ git branch -d dev

删除后，查看branch，就只剩下master分支了：

$ git branch
* master

以下是关于分支所经常使用的操作指令：

1.查看分支：git branch
2.创建分支：git branch <name>
3.切换分支：git checkout <name>
4.创建+切换分支：git checkout -b <name>
5.合并某分支到当前分支：git merge <name>
6.删除分支：git branch -d <name>
